Git gitignore

匹配规则

  • 以斜杠/开头表示目录。
  • 以星号*通配多个字符。
  • 以问号?通配单个字符。
  • 以方括号[]包含单个字符的匹配列表。
  • 以叹号!表示不忽略（跟踪）匹配到的文件或目录。

此外，git 对于 .ignore 配置文件是按行从上到下进行规则匹配的，意味着如果前面的规则匹配的范围更大，则后面的规则将不会生效。

强制添加到Git

.gitignore忽略规则查看

Git会告诉我们，.gitignore的第3行规则忽略了该文件，于是我们就可以知道应该修订哪个规则。
